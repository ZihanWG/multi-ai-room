# 行为准则 / Code of Conduct

## 我们的标准 / Our Standards

这个项目应该是一个务实、尊重彼此的协作空间。我们期望的行为包括：

This project should be a practical and respectful place to collaborate. Expected behavior includes:

- 讨论代码和设计时直接、具体、建设性。
  Be direct, specific, and constructive when discussing code and design.
- 在信息不清楚时先提问澄清。
  Ask for clarification when something is unclear.
- 反馈聚焦在工作本身，而不是个人。
  Keep feedback focused on the work, not the person.
- 尊重隐私，不发布 API Key、私有日志、凭证或个人信息。
  Respect privacy and do not post API keys, private logs, credentials, or personal information.

不可接受的行为包括：

Unacceptable behavior includes:

- 骚扰、侮辱或歧视性语言。
  Harassment, insults, or discriminatory language.
- 未经允许发布他人的私人信息。
  Publishing someone else's private information without permission.
- 在 issue、discussion 或 pull request 中反复发布无关干扰内容。
  Repeated off-topic disruption of issues, discussions, or pull requests.
- 迫使维护者或贡献者披露敏感凭证或私人账号信息。
  Pressuring maintainers or contributors to disclose sensitive credentials or private account details.

## 执行 / Enforcement

维护者可以编辑、隐藏或删除违反这些标准的评论和 issue。重复或严重违规可能导致临时或永久限制参与项目。

Maintainers may edit, hide, or remove comments and issues that violate these standards. Repeated or severe violations may result in temporary or permanent restriction from participating in the project.

如果需要报告行为准则问题，请创建一个不包含敏感细节的最小公开 issue，并请求私下沟通渠道。

If you need to report a conduct issue, open a minimal public issue without sensitive details and ask for a private contact path.
